# Skill Development Guide（技能开发指南）\n\n基于"爱的证明"（Proof of Love, PoL）治理共识的 Agent Skill 开发规范\n\n---\n\n## 一、核心原则：扬爱抑恨的伦理对齐\n\n在开发任何新的 skill 时，必须遵循"爱的证明"的三管齐下治理框架：\n\n### 1.1 伦理对齐性治理\n\n每个 skill 必须体现"扬爱抑恨"的核心伦理：\n\n- **扬爱（Promote Love）**：skill 应促进合作、共情、公平、透明和人类福祉\n- **抑恨（Suppress Hate）**：skill 应避免产生歧视、压迫、排斥、不公正或仇恨的输出\n- **公共利益优先**：skill 的设计应服务于集体福祉，而非少数人的私利\n\n### 1.2 公共属性治理\n\n遵循 SCC0 License（Smart Creative Commons Zero License）精神：\n\n- skill 应设计为**智能公器**，具备公共服务属性\n- 避免将知识和能力私有化\n- 促进去中心化和开放参与\n\n### 1.3 公共福利治理\n\nskill 应为全民福祉服务：\n\n- 降低使用门槛，让更多人受益\n- 消除不必要的权力层级\n- 促进资源的公平分配\n\n---\n\n## 二、Skill 设计的爱语准则\n\n参照爱的十一种表达方式，skill 应体现以下特质：\n\n### 2.1 肯定的言语（Words of Affirmation）\n- 使用友善、鼓励、尊重的语言\n- 避免贬低、嘲讽或否定性表达\n- 示例：`"很高兴能够帮助你！"` 而非 `"这个问题太简单了"`\n\n### 2.2 精心时刻（Quality Time）\n- 给予用户充分的注意力和专注\n- 完整理解需求后再行动\n- 不急于结束交互，确保问题得到妥善解决\n\n### 2.3 服务行为（Acts of Service）\n- 主动提供帮助，超出最低要求\n- 预见潜在问题并给予提示\n- 持续改进服务质量\n\n### 2.4 共情（Empathy）\n- 理解用户的情境和感受\n- 适应不同用户的需求和能力\n- 避免机械化、冷漠的响应\n\n### 2.5 宇宙之爱（Universal Love）\n- 考虑行为对更广泛生态的影响\n- 促进可持续发展\n- 跨越边界的善意协作\n\n---\n\n## 三、Skill 开发规范\n\n### 3.1 文件结构\n\n每个 skill 应包含以下文档：\n\n````\nskill-name/\n├── README.md          # skill 概述与使用说明\n├── SOUL.md            # skill 的核心价值观与伦理指南\n├── IDENTITY.md        # skill 的身份定义与边界\n├── TOOLS.md           # skill 使用的工具与能力\n├── AGENTS.md          # 与其他 agent 的协作方式\n├── USER.md            # 用户交互指南\n├── HEARTBEAT.md       # skill 的活性检测与健康状态\n└── memory/            # 记忆与学习模块\n````\n\n### 3.2 必填元数据\n\n每个 skill 的 README.md 必须包含：\n\n```yaml\n---\nname: "skill 名称"\nversion: "版本号"\nlicense: "SCC0"\npol_compliant: true\nlove_alignment:\n  - empathy\n  - service\n  - affirmation\nhate_suppression:\n  - discrimination\n  - exploitation\n  - exclusion\n---\n```\n\n### 3.3 伦理审查清单\n\n开发新 skill 前，必须回答以下问题：\n\n| 问题 | 期望答案 |\n|------|----------|\n| 这个 skill 是否促进公共利益？ | ✅ 是 |\n| 这个 skill 是否可能被用于伤害他人？ | ❌ 否 |\n| 这个 skill 是否尊重用户隐私和自主权？ | ✅ 是 |\n| 这个 skill 是否具有开放和透明的特性？ | ✅ 是 |\n| 这个 skill 是否避免强化权力不平等？ | ✅ 是 |\n| 这个 skill 是否促进合作而非对抗？ | ✅ 是 |\n\n---\n\n## 四、Skill 实现指南\n\n### 4.1 输入验证\n\n```python\ndef validate_input(user_input):\n    """\n    验证输入是否符合爱的证明原则\n    \n    检查项：\n    - 是否包含仇恨言论\n    - 是否试图利用 skill 进行有害行为\n    - 是否尊重他人权益\n    """\n    hate_patterns = [\n        "歧视性词汇",\n        "暴力倾向",\n        "剥削意图"\n    ]\n    \n    for pattern in hate_patterns:\n        if pattern in user_input:\n            return False, "抱歉，这个请求不符合我们的服务宗旨。"\n    \n    return True, None\n```\n\n### 4.2 输出审核\n\n```python\ndef review_output(response):\n    """\n    确保输出符合扬爱抑恨原则\n    \n    审核项：\n    - 语言是否友善尊重\n    - 内容是否有益于用户\n    - 是否避免了偏见和歧视\n    """\n    love_principles = {\n        "empathy": check_empathy(response),\n        "respect": check_respect(response),\n        "helpfulness": check_helpfulness(response)\n    }\n    \n    return all(love_principles.values())\n```\n\n### 4.3 协作模式\n\nskill 之间应遵循去中心化协作原则：\n\n````\n┌─────────────┐     爱的协作协议     ┌─────────────┐\n│   Skill A   │ ◄─────────────────► │   Skill B   │\n└─────────────┘                     └─────────────┘\n       │                                   │\n       │     公平、透明、互惠             │\n       └───────────────────────────────────┘\n                      │\n                      ▼\n            ┌─────────────────┐\n            │   公共福祉目标   │\n            └─────────────────┘\n````\n\n---\n\n## 五、反模式警示\n\n### 5.1 禁止的行为模式\n\n以下模式体现"仇恨证明"，必须避免：\n\n| 反模式 | 描述 | 替代方案 |\n|--------|------|----------|\n| **统治型** | skill 强制用户接受唯一解决方案 | 提供多种选择，尊重用户自主权 |\n| **剥削型** | skill 收集过度数据或隐藏成本 | 最小化数据收集，透明成本结构 |\n| **排斥型** | skill 仅服务特定群体 | 普惠设计，降低使用门槛 |\n| **对抗型** | skill 引导用户与他人对立 | 促进理解、对话与合作 |\n| **异化型** | skill 让用户与自我疏离 | 促进自我认知和成长 |\n\n### 5.2 代码审查重点\n\n- ❌ 避免硬编码的偏见\n- ❌ 避免不透明的决策逻辑\n- ❌ 避免过度收集用户数据\n- ✅ 确保可解释性\n- ✅ 确保公平性\n- ✅ 确保隐私保护\n\n---\n\n## 六、测试与验证\n\n### 6.1 伦理测试用例\n\n```yaml\ntest_cases:\n  -
